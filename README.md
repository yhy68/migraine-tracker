## 项目介绍

本工具用于追踪每一次偏头痛发作的详细信息，包括疼痛程度、部位、类型、前驱症状、伴随症状、可能诱因及用药效果。通过长期记录，帮助用户和医生识别发作规律与触发因素，从而更好地预防和管理偏头痛。

**核心设计原则：**

- **隐私优先**：所有数据仅保存在用户浏览器本地和用户的私有 GitHub 仓库中，开发者无法访问。
- **离线优先**：本地 LocalStorage 为主数据源，GitHub 仅为备份同步通道。
- **零依赖**：纯静态 HTML/CSS/JS，无任何第三方框架或 CDN 依赖。
- **移动端优先**：响应式设计，手机端体验优先，同时适配平板和桌面。

## 功能详解

### 1. 记录发作（记录页）

**基础信息：**

| 字段 | 说明 |
|------|------|
| 发作时间 | 开始时间（必填）和结束时间（可选），使用独立的日期和时间选择器，时间格式为 24 小时制 |
| 疼痛程度 | 1–10 级可视化选择按钮，颜色渐进（绿→黄→橙→红） |
| 疼痛部位 | 多选芯片：左侧、右侧、双侧、前额、后脑、眼眶/太阳穴、头顶、后颈、下颌/脸颊、全头 |
| 疼痛类型 | 单选：搏动性、压迫性、刺痛、钝痛、电击样、撕裂感、烧灼感 |
| 用药记录 | 默认预填「喜适美（佐米曲普坦口腔崩解片）2.5mg」，可添加多行、多选药品、填写剂量/用药时间/效果评分 |

**更多细节（展开可见）：**

- **前驱症状**：视觉模糊、闪光/暗点、频繁打哈欠、疲劳、肢体麻木、言语困难
- **伴随症状**：恶心、呕吐、畏光、畏声、头晕、颈部僵硬、头皮触痛
- **可能诱因**：压力/焦虑、睡眠不足、睡眠过多、天气变化、特定食物、酒精/红酒、咖啡因、饥饿/不规律进食、强光/闪烁、刺激性气味、噪音、经期/激素、排卵期、屏幕时间过长、姿势/颈椎问题、脱水/饮水不足、剧烈运动、巧克力/奶酪、空调/冷风直吹、情绪波动、气压/海拔变化
- **备注**：自由文本，可记录特殊饮食、天气、经期、情绪变化等

**交互特性：**

- 保存后自动重置表单，方便连续记录
- 支持编辑已有记录（从历史页点击「编辑」进入）
- 保存后可配置为自动同步到 GitHub

---

### 2. 历史记录（历史页）

**筛选功能：**

- **快捷筛选芯片**：全部 / 最近7天 / 最近30天 / 最近90天 / 自定义范围
- **自定义范围**：点击「自定义范围」展开日期选择行，选择开始和结束日期后点击「确定」
- **搜索**：实时搜索备注字段内容

**分页功能：**

- 默认每页显示 **5 条**最新记录
- 可通过「每页」下拉框切换：**5条 / 10条 / 20条 / 50条 / 全部**
- 分页控件在超过 1 页时自动显示
- 支持上一页/下一页及页码跳转
- 超过 7 页时自动显示省略号，保持界面整洁

**记录卡片：**

- 显示日期、持续时间、疼痛程度（彩色标签）
- 展开可见完整信息：类型、前驱症状、伴随症状、诱因、用药记录、备注
- 支持编辑和删除操作

---

### 3. 设置（设置页）

**数据备份：**

- **导出 JSON**：将全部记录导出为 JSON 文件，用于跨设备迁移或安全备份
- **导出 Excel**：导出为 `.xls` 文件（UTF-8 BOM CSV 格式），可用 Excel 直接打开
- **导入 JSON**：从 JSON 文件导入记录，自动与本地记录合并（以最新更新时间为准）

**手动同步：**

- 点击「立即同步」按钮，手动触发与 GitHub 私有仓库的双向同步
- 同步状态实时显示：已同步 / 同步中... / 同步失败
- 同步状态旁显示最近一次成功同步的时间

**主题设置：**

- **切换主题**：手动在浅色/深色之间切换
- **自动模式**：根据当前时间自动切换（19:00–次日7:00 为深色，7:00–19:00 为浅色），每分钟自动检查
- 当前模式提示文字实时更新

**其他设置：**

- **自动同步**：开启后，每次保存记录时自动同步到 GitHub
- **隐私声明**：可重新显示首页的隐私声明横幅

## 使用指南

### 首次使用

1. 打开应用地址：https://yhy68.github.io/migraine-tracker/
2. 阅读隐私声明（仅首次显示，可随时在设置中重新显示）
3. 开始记录第一次发作

### 记录发作

1. 在「记录」页填写发作信息
2. 必填项：开始时间、疼痛程度
3. 可选项：结束时间、疼痛部位、疼痛类型、用药记录、更多细节
4. 点击「保存记录」

1. 切换到「历史」页
2. 使用筛选芯片快速筛选记录
3. 使用搜索框搜索备注内容
4. 点击记录卡片展开查看完整信息
5. 点击「编辑」修改记录，点击「删除」删除记录

### 数据备份

1. 切换到「设置」页
2. 点击「导出 JSON」或「导出 Excel」备份数据
3. 如需导入，点击「导入 JSON 文件」选择备份文件

## 技术架构

### 技术栈

- **前端**：纯 HTML5 + CSS3 + JavaScript（ES6+）
- **存储**：LocalStorage（本地） + GitHub API（云端备份）
- **部署**：GitHub Pages
- **依赖**：零依赖，无任何第三方库

### 文件结构

```
migraine-tracker/
├── index.html              # 主页面
├── css/
│   └── style.css          # 全局样式（含 CSS 变量、响应式、主题）
├── js/
│   ├── app.js             # 主应用逻辑（UI 渲染、交互、状态管理）
│   └── storage.js         # 数据持久化层（LocalStorage + GitHub API）
├── README.md              # 项目文档
└── assets/               # 静态资源（如有）
```

### 数据同步机制

1. **本地优先**：所有操作（新增/编辑/删除）先写入 LocalStorage
2. **自动同步**（可选）：保存记录后自动触发 `Storage.syncData()`
3. **手动同步**：用户点击「立即同步」按钮
4. **合并策略**：以 `updatedAt` 时间戳为准，较新的记录覆盖较旧的记录
5. **冲突处理**：同一记录在本地和远程都有更新时，保留最新版本

### 主题系统

- 使用 CSS 自定义属性（CSS Variables）实现主题切换
- 浅色主题：默认，无 `data-theme` 属性
- 深色主题：设置 `document.documentElement.setAttribute('data-theme', 'dark')`
- 自动模式：不保存用户选择，每分钟检查当前时间并自动切换

## 数据结构

### 记录对象（Record）

```javascript
{
  "id": "string",              // 唯一标识（Date.now().toString(36) + 随机串）
  "version": 1,               // 数据版本号
  "createdAt": "ISO 8601",   // 创建时间
  "updatedAt": "ISO 8601",   // 最后更新时间
  "startTime": "ISO 8601",   // 发作开始时间（必填）
  "endTime": "ISO 8601",     // 发作结束时间（可选）
  "painLevel": 1-10,         // 疼痛程度
  "painLocations": ["left"],  // 疼痛部位（多选）
  "painType": "throbbing",    // 疼痛类型（单选）
  "aura": ["flashes"],        // 前驱症状（多选）
  "symptoms": ["nausea"],     // 伴随症状（多选）
  "triggers": ["stress"],     // 可能诱因（多选）
  "medications": [            // 用药记录（数组）
    {
      "name": "喜适美",
      "dose": "2.5mg",
      "time": "HH:MM",
      "effect": 3
    }
  ],
  "notes": "string"           // 备注（可选）
}
```

### LocalStorage 键名

| 键名 | 说明 |
|------|------|
| `migraine_records` | 本地记录数组（JSON 字符串） |
| `migraine_config` | GitHub 配置（用户名、令牌、仓库名） |
| `theme` | 用户手动选择的主题（`'light'` / `'dark'`，不存在则为自动模式） |
| `auto_sync` | 自动同步开关（`'1'` / `'0'`） |
| `lastSyncTime` | 最近一次成功同步的时间戳（毫秒） |
| `lastSyncStatus` | 最近一次同步状态（`'synced'` / `'error'`） |
| `privacy_dismissed` | 隐私声明是否已关闭（`'1'`） |

## 开发规范

### 代码风格

- **缩进**：2 空格
- **引号**：单引号（JavaScript 字符串）
- **命名**：
  - 变量/函数：`camelCase`
  - 常量：`UPPER_SNAKE_CASE`
  - CSS 类名：`kebab-case`
  - HTML `id`/`data-*`：`kebab-case`
- **注释**：函数上方使用单行或块注释说明用途

### 架构规范

- **关注点分离**：
  - `app.js`：负责 UI 渲染、用户交互、应用状态
  - `storage.js`：负责数据持久化、GitHub API 通信、数据合并逻辑
  - `style.css`：负责所有视觉样式、主题变量、响应式布局
- **模块化**：使用 IIFE（立即执行函数表达式）封装模块，通过 `return` 暴露公共 API
- **状态管理**：应用状态保存在 `app.js` 的闭包变量中，持久化状态保存在 LocalStorage

### 新增功能流程

1. 在 `storage.js` 中添加数据操作函数（如需新的数据操作）
2. 在 `app.js` 中添加渲染函数和事件处理函数
3. 在 `style.css` 中添加对应样式
4. 将新函数加入模块的公共 API（`return` 对象）
5. 在 HTML 中绑定事件（或使用 `onclick` 属性）
6. 测试移动端和桌面端的显示效果
7. 提交代码并推送到 GitHub

### 禁止事项

- **禁止引入第三方 CDN 依赖**（保持零依赖原则）
- **禁止将敏感信息硬编码**（GitHub Token 已做基础混淆，但仍需谨慎）
- **禁止直接操作 DOM 以外的全局状态**（统一通过 LocalStorage 或闭包变量）
- **禁止在生产环境中 `console.log` 输出敏感数据**

## 部署说明

### GitHub Pages 部署

1. 将代码推送到 `yhy68/migraine-tracker` 仓库的 `main` 分支
2. 在仓库设置中启用 GitHub Pages，选择 `main` 分支作为源
3. 访问 `https://yhy68.github.io/migraine-tracker/` 查看效果

### 数据仓库配置

应用使用独立的私有仓库 `yhy68/migraine-data` 存储记录数据：

- 仓库必须是**私有的**
- 数据文件路径：`data/records.json`
- GitHub Personal Access Token 需要 `repo` 权限
- Token 已做基础混淆处理（防止爬虫直接提取），但仍建议定期更换

### 本地开发

直接用浏览器打开 `index.html` 即可，无需任何构建步骤或本地服务器。
